if something is being defined before its declaration, this is called hoisting, and is done using var only(not let)
Example-
    console.log(a);
    a=11;
    console.log(a)
    var a
• var is hoisted and initialized with undefined.
• let and const are hoisted but placed in the Temporal Dead Zone (TDZ) until their declaration line is executed.

Hoisting in JavaScript is the behavior where declarations are processed before code execution, making them appear as if they are “moved” to the top of their scope.

Here’s the essential breakdown:
• Variable and function declarations are hoisted.
• Initializations are NOT hoisted.
• var is hoisted and initialized with undefined.
• let and const are hoisted but placed in the Temporal Dead Zone (TDZ) until their declaration line is executed.
• Function declarations are fully hoisted (you can call them before they appear in code).
• Function expressions are NOT hoisted like functions — only the variable declaration is.
